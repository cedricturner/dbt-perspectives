The stored procedure is not your enemy

Adoption & Culture

Data teams don't have a quality problem. They have a trust problem.

Tests tell you when something broke. They don't make the pipeline legible. Most teams are rebuilding the same model five times — not because the first was broken, but because they couldn't tell whether it was right.

Architecture

Python or SQL is the wrong question

The language doesn't matter. Where the computation runs does. When you transform data in Python, you're solving a different problem than language preference — and creating a different set of constraints.

Data Engineering

What moving to dbt actually changes

It's not the SQL syntax, version control, or modular files. It's that the pipeline stops disappearing when it runs. Every intermediate step becomes a table. That changes everything about how you debug, trust, and build on top of it.
